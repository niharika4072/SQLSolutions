Triggers in PL/SQL

A Trigger in PL/SQL is a stored procedure that automatically executes or fires when a specific event occurs in the database. Triggers are useful for enforcing business rules, validating input data, maintaining audit trails, and preventing invalid transactions.

Types of Triggers

  • BEFORE Trigger - Executes before the triggering event (INSERT, UPDATE, DELETE).
  • AFTER Trigger - Executes after the triggering event.
  • INSTEAD OF Trigger - Used with views to perform INSERT, UPDATE, DELETE operations.

Syntax of Trigger

CREATE [OR REPLACE] TRIGGER trigger_name
{BEFORE | AFTER | INSTEAD OF} triggering_event
ON table_name
[FOR EACH ROW]
BEGIN
-- Trigger logic
END;

Example 1: BEFORE INSERT Trigger

This trigger automatically sets the joining date to the current date when a new employee is added.

CREATE OR REPLACE TRIGGER set_join_date
BEFORE INSERT ON employees
FOR EACH ROW
BEGIN
:NEW.join_date := SYSDATE;
END;

Example 2: AFTER DELETE Trigger

This trigger stores deleted records in the audit table after deletion from employees table.

CREATE OR REPLACE TRIGGER audit_delete
AFTER DELETE ON employees
FOR EACH ROW
BEGIN
INSERT INTO employees_audit (emp_id, deleted_date)
VALUES (:OLD.emp_id, SYSDATE);
END;
