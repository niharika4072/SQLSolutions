Exception Handling in PL/SQL

Exception handling in PL/SQL is a powerful mechanism to handle runtime errors in a structured and controlled way. Exceptions are errors or unusual conditions that disrupt the normal flow of the PL/SQL program. PL/SQL provides built-in support for handling these exceptions gracefully without terminating the program abruptly.

When an exception occurs, PL/SQL transfers the control to the exception handling section of the program. This helps in maintaining the stability, accuracy, and reliability of the code.

Types of Exceptions in PL/SQL

  • Predefined Exceptions - Built-in exceptions provided by Oracle (e.g., NO_DATA_FOUND, TOO_MANY_ROWS, ZERO_DIVIDE)
  • User-defined Exceptions - Declared explicitly by the user in the DECLARE section.

Syntax of Exception Handling in PL/SQL

DECLARE
  -- variable declarations
  -- user-defined exception declaration (optional)
BEGIN
  -- executable statements
EXCEPTION
  WHEN exception_name1 THEN
    -- handling code
  WHEN exception_name2 THEN
    -- handling code
  WHEN OTHERS THEN
    -- handling code for all other exceptions
END;

1. Predefined Exceptions in PL/SQL

Oracle provides many predefined exceptions that cover common errors.

Common Predefined Exceptions

  • NO_DATA_FOUND - Raised when a SELECT INTO statement returns no rows.
  • TOO_MANY_ROWS - Raised when a SELECT INTO statement returns more than one row.
  • ZERO_DIVIDE - Raised when a number is divided by zero.
  • INVALID_NUMBER - Raised when conversion of character string to number fails.

Example of Predefined Exception

DECLARE
  v_name employees.employee_name%TYPE;
BEGIN
  SELECT employee_name INTO v_name FROM employees WHERE employee_id = 500;
  DBMS_OUTPUT.PUT_LINE('Employee Name: ' || v_name);
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    DBMS_OUTPUT.PUT_LINE('No such employee found.');
END;

2. User-Defined Exception

PL/SQL allows the user to create their own exceptions to handle custom error conditions.

Steps for User-Defined Exception

  • Declare the Exception
  • Raise the Exception using RAISE keyword
  • Handle the Exception in EXCEPTION section

Example of User-Defined Exception

DECLARE
  low_salary EXCEPTION;
  v_salary employees.salary%TYPE;
BEGIN
  SELECT salary INTO v_salary FROM employees WHERE employee_id = 101;
  IF v_salary < 5000 THEN
    RAISE low_salary;
  END IF;
  DBMS_OUTPUT.PUT_LINE('Salary is sufficient.');
EXCEPTION
  WHEN low_salary THEN
    DBMS_OUTPUT.PUT_LINE('Salary is below the minimum required.');
END;

3. WHEN OTHERS Clause

The WHEN OTHERS clause is used to handle all other exceptions that are not explicitly handled. It should be written at the end of the EXCEPTION section.

Example of WHEN OTHERS

BEGIN
  -- Some PL/SQL code
EXCEPTION
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('Cannot divide by zero.');
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('An unknown error occurred.');
END;

Advantages of Exception Handling in PL/SQL

  • Improves code reliability and robustness.
  • Provides meaningful error messages to the user.
  • Maintains the normal flow of program execution even after errors.
  • Makes debugging and troubleshooting easier.
