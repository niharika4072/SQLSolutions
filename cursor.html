Cursor Management in PL/SQL

In PL/SQL, a cursor is a pointer that allows you to handle query result sets one row at a time. When a SELECT statement returns multiple rows, cursors are used to fetch and process each row individually. Without cursors, it would be difficult to handle multi-row query results in PL/SQL.

There are two types of cursors in PL/SQL:

  • Implicit Cursor
  • Explicit Cursor

1. Implicit Cursor

PL/SQL automatically creates an implicit cursor whenever a SELECT statement returns a single row or DML operations like INSERT, UPDATE, DELETE are executed.

Important Attributes of Implicit Cursor

  • %FOUND - Returns TRUE if DML operation affects one or more rows.
  • %NOTFOUND - Returns TRUE if DML operation affects no rows.
  • %ROWCOUNT - Returns the number of rows affected.
  • %ISOPEN - Always FALSE for implicit cursors because they close automatically.

Example of Implicit Cursor

BEGIN
  UPDATE employees SET salary = salary + 500 WHERE department_id = 10;
  IF SQL%FOUND THEN
    DBMS_OUTPUT.PUT_LINE(SQL%ROWCOUNT || ' rows updated.');
  ELSE
    DBMS_OUTPUT.PUT_LINE('No rows updated.');
  END IF;
END;

2. Explicit Cursor

Explicit cursors are declared by the programmer when a SELECT statement returns more than one row and needs to be processed row-by-row.

Steps to Use Explicit Cursor

  • Declare the Cursor
  • Open the Cursor
  • Fetch Data from Cursor
  • Close the Cursor

Syntax of Explicit Cursor

DECLARE
  CURSOR cursor_name IS SELECT_statement;
BEGIN
  OPEN cursor_name;
  FETCH cursor_name INTO variable_list;
  CLOSE cursor_name;
END;

Example of Explicit Cursor

DECLARE
  CURSOR emp_cursor IS SELECT employee_id, employee_name FROM employees;
  v_id employees.employee_id%TYPE;
  v_name employees.employee_name%TYPE;
BEGIN
  OPEN emp_cursor;
  LOOP
    FETCH emp_cursor INTO v_id, v_name;
    EXIT WHEN emp_cursor%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE('ID: ' || v_id || ' Name: ' || v_name);
  END LOOP;
  CLOSE emp_cursor;
END;

Cursor Attributes in Explicit Cursor

  • cursor_name%FOUND - Returns TRUE if a successful fetch occurred.
  • cursor_name%NOTFOUND - Returns TRUE if no more rows to fetch.
  • cursor_name%ROWCOUNT - Returns number of rows fetched so far.
  • cursor_name%ISOPEN - Returns TRUE if the cursor is open.

3. Cursor FOR LOOP (Simplified Method)

PL/SQL provides Cursor FOR LOOP to automatically open, fetch, and close the cursor.

Syntax of Cursor FOR LOOP

FOR record_variable IN cursor_name LOOP
  statements;
END LOOP;

Example of Cursor FOR LOOP

DECLARE
  CURSOR emp_cursor IS SELECT employee_id, employee_name FROM employees;
BEGIN
  FOR emp_rec IN emp_cursor LOOP
    DBMS_OUTPUT.PUT_LINE('ID: ' || emp_rec.employee_id || ' Name: ' || emp_rec.employee_name);
  END LOOP;
END;

Advantages of Cursor FOR LOOP

  • No need to declare variables for fetching data.
  • No need to explicitly open or close the cursor.
  • Automatically handles fetching of rows.
